Jimple Content Outline View

This view is where you will see the content outline for Jimple files. It follows the concept of the Java Outline View.

It only (at this point) gives visual indications of public, protected, private fields, methods and classes and interfaces. It does not indicate abstract, final, static, synchronized, volatile, native or transient.

It also does not update dynamically, you must close and re-open the Jimple file to get an accurate outline and if you edit the Jimple file and then select an element in the outline it may be incorrect.

Actions Available

There are no actions available from this view.
